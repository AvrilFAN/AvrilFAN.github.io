Routage Inter-VLAN (Router-on-a-Stick)

Concept Clé

Router-on-a-Stick(Routage sur un bâton)

Rôle Priincipal: Permettre à des ordinateurs situés dans des VLANs et des sous-réseaux différents de communiquer entre eux. On utilise pour cela un seul câble physique entre le switch et le routeur grâce à la création de “sous-interfaces” virtuelles.

Le Trunk(Protocole 802.1Q)

Rôle Priincipal: Ajouter une étiquette(un tag) sur les données pour identifier leur VLAN d’origine. Cela permet de faire passer les données de plusieurs VLANs différents sur un seul câble sans les mélanger.

Cisco - Configuration du routage entre deux VLANs

Objectif de l’exercise

Connecter un Router et un Switch avec seurlement un câble pour faire communiquer entre deux différents VLANs qui ne sont pas dans le méme sous-réseau.

Etapes clés de configuration

Installation du matériel et des adresses IP

  1. Connexion: Brancher le PC1 sur le port Fa0/1 du switch, le PC0 sur le port Fa0/2, et relier le port Fa0/3 du switch au port Gig0/0 du routeur.
  2. Configuration des PC:
    PC1 (VLAN 10) : IP = 192.168.10.10 / Passerelle (Gateway) = 192.168.10.1
    PC0 (VLAN 20) : IP = 192.168.20.10 / Passerelle (Gateway) = 192.168.20.1

Configuration du Commutateur (Switch0)

  1. Créer les VLANs : Mettre le port Fa0/1 dans le VLAN 10 et le port Fa0/2 dans le VLAN 20 (mode Access).
  2. Activer le Trunk : Configurer le port Fa0/3 (connecté au routeur) en mode Trunk pour accepter le trafic de tous les VLANs.

Configuration du Routeur (Router0 en ligne de commande CLI)

  1. Allumer le port : Entrer sur le port physique Gig0/0 et faire un no shutdown (ne pas mettre d’adresse IP ici).

Guide de dépannage (Troubleshooting)

Erreur 1 : Cliquer l’enveloppe directement sur le routeur

Problème: Quand on met l’enveloppe de test sur le routeur, Packet Tracer affiche l’erreur : Router0 has no functional ports.

Pourquoi ?: Le routeur possède plusieurs sous-interfaces virtuelles. Le logiciel ne sait pas à quelle interface (VLAN 10 ou 20) vous voulez envoyer le message.

Solution: Pour tester le fonctionnement, vous devez toujours cliquer sur le PC1 (Source) puis sur le PC0 (Destination). Ne cliquez jamais sur le routeur

Erreur 2 : Impossible de configurer le routeur avec l’interface graphique (GUI)

Problème : Dans l’onglet “Config” du routeur, il n’y a aucun bouton pour créer les sous-interfaces (.10 ou .20) ni pour choisir l’encapsulation dot1Q.
Pourquoi ? : L’interface graphique de Packet Tracer est très limitée. Elle est faite pour les configurations de base. Dans le monde réel, un administrateur réseau utilise toujours la ligne de commande (CLI).
Solution : Il faut utiliser le mode CLI. Entrez en mode de configuration globale (Router(config)#) et collez les lignes de code données à l’étape 3. Quand vous voyez le message changed state to up, cela signifie que les connexions virtuelles fonctionnent.
